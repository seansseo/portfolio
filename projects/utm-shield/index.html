UTM Shield

A Chrome extension that reads, validates, and QAs UTM parameters in real time. Catch broken tracking before campaigns go live, not after the data is already wrong.

The Problem

UTM parameters are the backbone of campaign attribution. Every marketing team relies on them to connect ad spend to conversions. But they break constantly: typos in source names, inconsistent capitalization, missing parameters, deprecated values that nobody updated.

The result is dirty data. Google Analytics reports fragment across misspelled campaign names. Attribution models fall apart. Teams make budget decisions based on numbers that don't reflect reality. And the worst part is that nobody notices until weeks later, when the damage is already done.

Most teams try to solve this with naming conventions in a spreadsheet. It works until someone forgets. There's no validation at the point where it matters: the moment a URL goes live.

The Solution

UTM Shield lives in the browser where marketers already work. It reads UTM parameters from any URL on the current page and runs them through a validation engine that checks for common issues: missing required fields, inconsistent naming conventions, known typos, and deprecated values.

Validation results appear inline with clear pass/fail indicators. Issues are flagged with specific, actionable messages rather than vague errors. The goal is to catch problems at the point of creation, not in a post-campaign audit.

For teams, UTM Shield supports shared naming conventions so everyone validates against the same rules. No more "was it lowercase or title case for the source?" questions.

Key Features

UTM Reader

Parses and displays UTM parameters from any URL on the current page. See exactly what's being tracked at a glance.

Validation Engine

Rules-based validation for required fields, naming patterns, capitalization consistency, and known issues.

QA Dashboard

Summary view of all UTM parameters found on a page with pass/fail status. Export results for team review.

Tech Stack

Chrome Extension API JavaScript UTM Validation

Where It Fits

UTM Shield is the Channel Layer of the 3-Layer Marketing Stack. The Data Layer (DocWeb) structures information. The Intelligence Layer processes it. The Channel Layer is where marketing meets the real world, and where QA matters most.

Bad UTM parameters don't just create messy reports. They break the entire feedback loop between spend and results. UTM Shield ensures that the data entering your analytics platform is clean from the start, so every layer above it can be trusted.
