The Problem

LLMs hallucinate. Ask one about a specific API parameter, CLI flag, or configuration schema and there's a good chance it'll confidently give you something that doesn't exist. The training data is often outdated or generic â€” and for technical documentation, "close enough" isn't good enough.

The fix is grounding: feed the model official, up-to-date documentation so it has the real source of truth. Tools like NotebookLM make this easy â€” if you have the URLs. But most documentation sites have hundreds or thousands of pages spread across nested sitemaps. Manually hunting for the right URLs is tedious and incomplete.

I kept running into this friction every time I wanted to create an AI-powered reference for a new technology. The Documentation Extractor was built to eliminate it.

The Solution

Enter a domain and the Documentation Extractor discovers its sitemaps â€” checking robots.txt for directives and falling back to common paths. It recursively parses every sitemap index, fetches URLs in batches with exponential-backoff retry, and deduplicates the results.

A keyword filtering UI lets you narrow results in real time â€” include or exclude URL paths with a tag-bubble interface â€” then copy the filtered list to your clipboard with one click. Paste directly into NotebookLM and you've got an expert guide grounded in official sources, built in seconds instead of hours.
